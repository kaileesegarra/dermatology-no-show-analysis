1. Question and Motivation¶

Missed medical appointments are a practical healthcare problem and a useful data science problem. In clinics, no-shows create unused appointment slots, longer wait times, delayed care, and inefficiencies for providers and staff. From a patient perspective, missing an appointment can delay diagnosis, treatment, medication refills, or follow-up care. This is especially important in outpatient specialties such as dermatology, where patients may be monitoring changing lesions, chronic inflammatory skin disease, post-procedure healing, or medication side effects.

The central question for this tutorial is:

Can appointment scheduling, reminder, demographic, and neighborhood-level variables help predict whether a patient will miss a medical appointment?

This question is appropriate for data science because the outcome is measurable, binary, and operationally meaningful. Each appointment is labeled as either attended or missed, allowing this project to frame no-show prediction as a classification task. The goal is not to replace human judgment or make punitive decisions about patients. Instead, the model can help identify patterns that may support better scheduling workflows, reminder strategies, and patient outreach.

This final tutorial builds on the exploratory work from Milestones 1 and 2. Peer and instructor feedback emphasized that the project needed to move beyond EDA and clearly show model implementation, testing, and results. Therefore, the final version uses EDA to motivate the choice of predictors and then tests whether those predictors can support a logistic regression classifier.

From a data science perspective, this problem naturally fits into a supervised classification framework, where historical appointment data can be used to predict future attendance outcomes and support real-world decision-making.

4. Data Cleaning and Feature Engineering¶

The raw dataset is already structured as one row per appointment, which is a tidy format for this project. However, several transformations are necessary before analysis and modeling. Column names are standardized, date columns are converted to datetime values, the target variable is converted into a binary numeric variable, and new scheduling features are created.

The most important engineered feature is lead_time_days, which measures the number of days between when the appointment was scheduled and when it occurred. This feature is motivated by the EDA finding that longer scheduling delays appear associated with higher no-show rates.

In [5]:
# Standardize column names
clean_df = df.copy()
clean_df.columns = (
    clean_df.columns
    .str.strip()
    .str.lower()
    .str.replace('-', '_')
)

# Convert dates
clean_df['scheduledday'] = pd.to_datetime(clean_df['scheduledday'])
clean_df['appointmentday'] = pd.to_datetime(clean_df['appointmentday'])

# Binary target: 1 = missed appointment, 0 = attended appointment
clean_df['no_show_flag'] = (clean_df['no_show'] == 'Yes').astype(int)

# Appointment lead time in days
clean_df['lead_time_days'] = (
    clean_df['appointmentday'].dt.date.astype('datetime64[ns]') -
    clean_df['scheduledday'].dt.date.astype('datetime64[ns]')
).dt.days

# Replace impossible negative lead times with missing values
clean_df.loc[clean_df['lead_time_days'] < 0, 'lead_time_days'] = np.nan

# Calendar features
clean_df['appointment_weekday'] = clean_df['appointmentday'].dt.day_name()
clean_df['scheduled_hour'] = clean_df['scheduledday'].dt.hour

# Age cleaning: remove biologically impossible ages
clean_df.loc[(clean_df['age'] < 0) | (clean_df['age'] > 115), 'age'] = np.nan

# Age groups for EDA
clean_df['age_group'] = pd.cut(
    clean_df['age'],
    bins=[0, 18, 35, 60, 115],
    labels=['0-18', '19-35', '36-60', '60+'],
    include_lowest=True
)

# Delay groups for EDA
clean_df['delay_group'] = pd.cut(
    clean_df['lead_time_days'],
    bins=[-1, 0, 3, 7, 14, 30, clean_df['lead_time_days'].max()],
    labels=['Same day', '1-3 days', '4-7 days', '8-14 days', '15-30 days', '31+ days']
)

6. Model Selection: Why Logistic Regression?¶

The final model uses logistic regression because the response variable is binary: an appointment is either missed or attended. Logistic regression is a strong first model for this project because it is interpretable, appropriate for classification, and commonly used in healthcare and social science settings where explanation matters.

A more complex model such as random forest could potentially improve predictive performance, but the goal of this tutorial is not only prediction. The goal is also to explain which variables are associated with no-show risk. Logistic regression makes it easier to connect model results back to the EDA and to discuss coefficients, odds, and operational recommendations.

The model uses a train/test split so that evaluation is performed on data the model did not see during training. This directly addresses the instructor feedback that the final notebook needs implementation and prediction results, not only a model proposal.

While performance is moderate, the model prioritizes interpretability, which is critical in healthcare decision-making contexts.

Interpreting the Metrics¶

Accuracy alone can be misleading because most patients attend their appointments. A model that predicts every patient will attend could achieve a superficially high accuracy while failing to identify missed appointments. For this reason, precision, recall, F1 score, and ROC-AUC are included.

For this project, recall is especially important because it measures how many true no-shows the model successfully identifies. In a clinic workflow, missing a high-risk appointment may mean losing the chance to send an extra reminder or offer additional support. Precision is also important because too many false positives could waste staff time. The best operational model would balance recall with practical outreach capacity.

7. Feature Interpretation¶

Because logistic regression is interpretable, the coefficients can help explain how features contribute to predicted no-show risk. Positive coefficients increase the predicted log-odds of a no-show, while negative coefficients decrease the predicted log-odds. Since numeric features were standardized, coefficient magnitudes are more comparable for age and lead time.

8. Conclusions and Recommendations¶

This project shows that medical appointment no-shows can be studied as both an operational healthcare issue and a predictive data science problem. The exploratory analysis suggests that scheduling delay, age, SMS reminder status, weekday, and health/social indicators are associated with appointment attendance. The logistic regression model then tests whether these variables can be used together to predict no-show risk on unseen test data.

The most important conclusion is that scheduling delay should be treated as a major operational signal. Appointments scheduled far in advance may require additional support because patients have more time to forget, experience schedule conflicts, lose transportation, or become disconnected from the original reason for the visit. This does not mean long-delay patients are irresponsible; it suggests that clinic systems may need stronger follow-up workflows for appointments booked weeks in advance.

A second conclusion is that SMS reminders should be interpreted carefully. If the EDA shows that SMS recipients have higher no-show rates, this should not be described as proof that SMS reminders are ineffective. A more reasonable explanation is that SMS reminders may be associated with patients or appointments that already have higher risk. This is why the model includes SMS along with other variables rather than interpreting the SMS graph in isolation.

The model should be used as a decision-support tool, not as a final authority. A clinic could use predicted risk scores to prioritize extra reminder calls, transportation-resource information, earlier waitlist offers, or flexible rescheduling messages. The ethical goal should be improving access and reducing missed care, not penalizing patients.

Limitations¶

This analysis is observational, so it identifies associations rather than causal effects. The dataset comes from a Brazilian healthcare setting, so results may not generalize directly to every outpatient clinic or specialty. The supplemental socioeconomic dataset from Milestone 2 was manually constructed and too small to support strong claims unless it is expanded and made downloadable. Future work could improve the model by adding distance to clinic, appointment type, provider specialty, insurance/access variables, reminder timing, and prior attendance history.

From an operational standpoint, clinics could implement several practical strategies based on these findings. For example, patients with longer scheduling delays could be flagged for additional reminders or follow-up contact closer to the appointment date. Clinics could also experiment with scheduling policies that reduce long lead times when possible, or offer waitlist opportunities to re-engage patients. More advanced systems could integrate predictive risk scores directly into scheduling dashboards, allowing staff to proactively manage high-risk appointments. These interventions align with findings in healthcare research, where predictive models have been shown to improve attendance rates and overall clinic efficiency when used as decision-support tools (Healthcare Study on Predictive No-Show Modeling).

Overall, this project demonstrates how even simple, interpretable models can support more efficient, patient-centered healthcare systems.
